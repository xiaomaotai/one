# 浅色模式侧边栏 Bug 修复需求

## 介绍

在浅色模式下打开侧边栏时，存在多个 UI 显示问题，导致用户体验不佳。

## 术语表

- **浅色模式**: 应用主题设置为 light
- **深色模式**: 应用主题设置为 dark
- **侧边栏**: 从左侧滑出的导航抽屉
- **系统通知栏**: 设备顶部的状态栏（包含时间、信号等）
- **主内容区**: 侧边栏右侧的主要内容显示区域
- **遮罩层**: 侧边栏打开时覆盖主内容的半透明背景

## 需求

### 需求 1: 浅色模式下系统通知栏文字可见性

**用户故事**: 作为用户，我想在浅色模式下打开侧边栏时，系统通知栏的文字清晰可见，以便我能看到时间和其他系统信息。

#### 接受标准

1. WHEN 用户在浅色模式下打开侧边栏 THEN 系统通知栏的文字颜色应该是深色，确保与浅色背景形成足够的对比度
2. WHEN 用户在浅色模式下打开侧边栏 THEN 系统通知栏背景应该与侧边栏背景颜色一致（白色）
3. WHEN 用户在深色模式下打开侧边栏 THEN 系统通知栏的文字颜色应该是浅色，背景应该是深色

### 需求 2: 浅色模式下主内容区域颜色一致性

**用户故事**: 作为用户，我想在浅色模式下打开侧边栏时，主内容区域的背景颜色与整体浅色主题一致，避免出现颜色不匹配的情况。

#### 接受标准

1. WHEN 用户在浅色模式下打开侧边栏 THEN 侧边栏右侧未被遮罩的主内容区域应该显示浅色背景（与浅色模式一致）
2. WHEN 用户在浅色模式下打开侧边栏 THEN 遮罩层应该是半透明的深色，不影响背景颜色的显示
3. WHEN 用户在深色模式下打开侧边栏 THEN 主内容区域应该显示深色背景

### 需求 3: 浅色模式下侧边栏关闭后主题恢复

**用户故事**: 作为用户，我想在浅色模式下关闭侧边栏后，页面的文字和背景颜色能够正确恢复到浅色模式的样式。

#### 接受标准

1. WHEN 用户在浅色模式下打开侧边栏后关闭它 THEN 系统通知栏应该恢复为浅色文字和白色背景
2. WHEN 用户在浅色模式下打开侧边栏后关闭它 THEN 主内容区域应该恢复为浅色背景
3. WHEN 用户在浅色模式下打开侧边栏后关闭它 THEN 页面所有文字应该清晰可见，颜色对比度足够
